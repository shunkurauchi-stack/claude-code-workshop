Chapter 1

Claude Code とは？

🖥️

ターミナルで動くAI

ブラウザ不要。コマンドラインから直接AIと対話しながらコードを書く

📁

プロジェクト全体を理解

ファイル読み書き・検索・実行まで自律的にこなす。コードベース全体を把握した上で回答する

⚡

Anthropic 製・Claude 搭載

ChatGPT と同じく LLM ベース。Anthropic が開発した Claude モデルを使用

CLI ツール コード生成 ファイル操作 エージェント型AI

Chapter 1

普通のAIチャットとの違い

ChatGPT / Claude.ai Claude Code
操作場所 ブラウザ ターミナル（VS Code内も可）
ファイルアクセス 手動でコピペが必要 自動でファイルを読み書き
コード実行 できない その場で実行・確認
プロジェクト理解 貼り付けた部分のみ プロジェクト全体を把握
作業の自律性 回答するだけ 複数ステップを自律実行

Chapter 2

インストールと初期設定

1

Node.js をインストール

v18 以上が必要。node -v で確認。未インストールなら nodejs.org からダウンロード

2

Claude Code をインストール

$ npm install -g @anthropic-ai/claude-code
3

プロジェクトフォルダで起動

$ cd my-project
$ claude
✓ Welcome to Claude Code! ...

Chapter 3

基本的な使い方 ①
指示の出し方

自然な日本語でOK。具体的に伝えるほど精度が上がる

あなたの指示
src/utils.ts に「日付を YYYY-MM-DD 形式に変換する関数」を追加して。既存の関数スタイルに合わせてください。
Claude Code の動作
1. src/utils.ts を読み込む
2. 既存スタイルを確認
3. 関数を追加・保存
4. 完了報告

Chapter 3

基本的な使い方 ②
チームで活かす機能

📋

スラッシュコマンド

/help でコマンド一覧表示
/clear で会話リセット
/compact でコンテキスト圧縮

🗂️

CLAUDE.md ― チーム共通設定

プロジェクトのルール・背景・コード規約をファイルに書くと毎回の説明が不要に。チームメンバー全員が同じ前提でAIを使える

🔑

権限管理（パーミッション）

ファイル削除や外部実行など危険な操作は確認を求めてくれる。信頼できる操作はあらかじめ許可設定も可能で安心して使える

📊

VS Code 統合

VS Code の拡張機能として使用可能。エディタを離れずにAIと対話しながら開発できる。既存フローをほぼ変えずに導入できる

Chapter 4

業務での活用事例

⚙️

新機能の実装

「〇〇機能を追加して」と自然言語で指示するだけで、関連ファイルを自動読み込みしてコードを生成・保存まで一気通貫で完了

🐛

バグ調査・修正

エラーログやスタックトレースを貼り付けると、コードベース全体を検索して原因を特定。修正案を提示して自動適用まで可能

📄

ドキュメント生成

既存コードを読み込んでREADME・API仕様書・コメントを自動生成。最新コードに合わせたドキュメント更新も素早くできる

🧪

テスト・コードレビュー支援

「この関数のテストを書いて」でユニットテストを自動生成。コードの問題点を指摘させるレビュー補助としても活用できる

デモタイム

ライブデモ

① 業務コードのバグを見つけて直す ② 仕様変更に合わせて関数を修正する ③ テストを自動生成する ④ コードのドキュメントを生成する

Tips

うまく使うコツ

✅

具体的に指示する

「直して」より「〇〇の関数のXXエラーを直して」。背景や制約も添えるとさらに精度が上がる

✅

小さなステップで進める

一度に多くを頼まず、1機能ずつ確認しながら進める。問題を早期に発見できる

✅

結果を必ず確認する

AIの変更をそのまま信頼せず、動作確認・レビューを忘れずに。最終責任は人間にある

⚠️

Git で管理する

変更前にコミット。間違えても戻せる状態で作業することで安心して使える

⚠️

機密情報に注意

APIキー・パスワード・個人情報はプロンプトに貼らない。.gitignore も事前に確認

⚠️

コスト感覚を持つ

使うほどトークンを消費。大きなファイルの一括読み込みや長い会話には注意
